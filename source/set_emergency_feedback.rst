============================
控制闸机紧急状态异步反馈
============================

场景描述
----------
1.服务端发送 `控制闸机紧急状态的指令 <set_emergency.html>`_ 后，闸机通过此接口异步反馈闸机的紧急状态。

2.按动闸机紧急按钮，闸机软件进入紧急状态，并将紧急状态推送至安检信息系统。

3.取消紧急后，恢复到紧急前的状态，并将正常状态推送至安检信息系统。

面向客户端
::::::::::::::::::::
闸机

请求URL
---------------------
**HTTP POST**  /api/gatefeedback/emergency

请求体格式
----------------------------
请求体格式为JSON，字段名不区分大小写::

    {
        "deviceCode":"string", //闸机设备编号
        "emergency":"int", //紧急状态, 1:紧急;0:正常
        "execute":"string" //执行结果说明
    }

请求体示例
----------------------------
::

    {
        "deviceCode":"Z001",
        "emergency":1,
        "execute":"OK"
    }
