============================
控制闸机等待状态异步反馈
============================

场景描述
----------
为实现安检流量控制，提示后续旅客等待，管理平台通过服务端会不定时地发送 `控制闸机等待安检(13)和恢复安检(14)的指令 <../manage/query_command.html>`_  后，闸机通过此接口异步反馈闸机的暂停状态。

面向客户端
::::::::::::::::::::
闸机

请求URL
---------------------
**HTTP POST**  /api/gatefeedback/pending

请求体格式
----------------------------
请求体格式为JSON，字段名不区分大小写::

    {
        "deviceCode":"string", //闸机设备编号
        "pending":"int", //等待状态, 1:等待;0:恢复
        "execute":"string" //执行结果说明
    }

请求体示例
----------------------------
::

    {
        "deviceCode":"Z001",
        "pending":1,
        "execute":"OK"
    }
