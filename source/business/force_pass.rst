====================
旅客从闸机二门放行
====================

场景描述
----------
当旅客因人脸比对失败超过上限或布控被锁定在安检闸机的一门和二门中间时，工作人员通过手持PAD输入授权码，
通过调用此接口实现从二门放行旅客，完成安检流程，形成业务闭环。

面向客户端
::::::::::::::::::::
移动管理平台

请求URL
---------------------
**HTTP POST**  /api/gate/forcepass

请求体格式
----------------------------
请求体格式为JSON，字段名不区分大小写::

    {
        "deviceCode":"string", //闸机设备编号
        "authorizeCode":"string" //授权码
    }

请求体示例
----------------------------
::

    {
        "deviceCode":"Z001",
        "authorizeCode":"1234"
    }
